AC13.01 | Utiliser un environnement de développement web

Ce que j’ai fait

Mise en place d'espaces de travail locaux structurés et utilisation d'outils de gestion de version pour coder des projets web de manière organisée.

Pourquoi je l’ai fait

Afin d'assurer le suivi des modifications du code source, d'éviter les pertes de données et de collaborer efficacement sur des fichiers d'application commune.

Comment je l’ai fait

Utilisation de l'éditeur Visual Studio Code couplé aux terminaux Git pour initialiser, valider (commit) et pousser (push) des dépôts de code sur un serveur distant.

$ git init
$ git add index.html style.css
$ git commit -m "Initialisation de l'interface et du style du portfolio"
$ git push origin main
Mes difficultés

Résoudre les conflits de fusion (merge conflicts) lors de modifications simultanées sur les mêmes lignes d'un fichier de style CSS.

Ce que j’en ai appris

J'ai assimilé la routine des workflows Git et compris l'importance absolue de commenter chaque commit pour maintenir un historique de projet transparent.

AC13.02 | Modéliser les données de l’application web

Ce que j’ai fait

Conception logique de l'organisation des données et modélisation de bases de données relationnelles pour stocker des informations de sites dynamiques.

Pourquoi je l’ai fait

Pour garantir la cohérence des bases de données de l'application, éliminer la redondance des informations et optimiser le temps d'exécution des futures requêtes.

Comment je l’ai fait

Création de dictionnaires de données, élaboration de modèles conceptuels (MCD) basés sur les dépendances fonctionnelles, puis transcription en requêtes SQL d'insertion.

CREATE TABLE Competences (
  id_comp INT AUTO_INCREMENT PRIMARY KEY,
  code_ac VARCHAR(10),
  titre_ac VARCHAR(255)
);
Mes difficultés

Déterminer de manière exacte les cardinalités complexes de type (0,N) ou (1,N) lors de la création de tables d'associations intermédiaires.

Ce que j’en ai appris

L'importance d'une analyse conceptuelle poussée en amont, qui évite de devoir restructurer une base de données en pleine phase de production applicative.

AC13.03 | Développer une application avec un langage de script

Ce que j’ai fait

Écriture de scripts serveurs et de programmes dynamiques pour traiter les données reçues, manipuler des fichiers et automatiser des actions logiques.

Pourquoi je l’ai fait

Afin de rendre les pages web interactives, de communiquer avec la base de données relationnelle et de générer du contenu personnalisé pour l'utilisateur.

Comment je l’ai fait

Utilisation du langage PHP pour récupérer les paramètres de formulaires ($_POST) et construction de requêtes préparées PDO sécurisées contre les failles.

$query = $pdo->prepare('SELECT * FROM Competences WHERE code_ac = :code');
$query->execute(['code' => $code_recherche]);
$resultat = $query->fetchAll();
Mes difficultés

Gérer la persistance des variables d'affichage à l'aide des sessions de connexion ($_SESSION) sans créer de failles d'élévation de droits.

Ce que j’en ai appris

J'ai compris l'importance critique de filtrer et de nettoyer systématiquement les saisies des internautes pour éviter les injections SQL destructrices.

AC13.05 | Traiter les données de l’application web

Ce que j’ai fait

Manipulation, filtrage et validation des flux de données transitant entre les formulaires utilisateurs et l'application serveur.

Pourquoi je l’ai fait

Afin de s'assurer de l'exactitude des types de données enregistrées (dates, adresses IP, emails) et d'éviter les anomalies de stockage.

Comment je l’ai fait

Mise en place d'expressions régulières (Regex) en langage de programmation et utilisation de fonctions natives de validation logique des chaînes.

if (filter_var($email_user, FILTER_VALIDATE_EMAIL)) {
    // Traitement logique de la donnée propre
}
Mes difficultés

Construire des motifs d'expressions régulières complexes sans rejeter par erreur des formats de saisie utilisateur pourtant valides.

Ce que j’en ai appris

Qu'un traitement rigoureux des données à la source allège considérablement la charge de nettoyage applicatif sur les systèmes de stockage finaux.
